# Layer 3 — 작성규칙 및 스타일 가이드

> **용도**: 사업계획서/제안서 작성 시 문장 스타일, 구조, 서식 규칙을 통일하기 위한 가이드  
> **사용법**: 문서 작성 시 본 파일의 규칙을 따를 것. Layer 1(팩트) + Layer 2(전략)과 함께 사용
> **최종 업데이트**: 2026.03.11

## 1. 문서 작성 워크플로우

### 1-1. Cowork 작업 시 프롬프트 템플릿

```
[기본 프롬프트]
다음 파일들을 참고하여 사업계획서를 작성해 주세요:

1. 00 공통 가이드/Layer1_코어팩트시트.md — 기술·데이터·IP 팩트
2. 00 공통 가이드/Layer2_전략_[플랫폼투트랙 or 투트랙 or 치료제단일].md — 전략 방향·사업모델
3. 00 공통 가이드/Layer3_작성규칙_스타일가이드.md — 문장 스타일·서식 규칙 (본 파일)
4. [사업 폴더]/00 가이드/[사업명]_지시사항.md — 해당 사업의 작성 지시·피드백·수정 원칙
5. [사업 폴더]의 원본 양식 — 목차·섹션 구조

작성 규칙:
- 원본 양식의 목차와 섹션 순서를 그대로 따를 것
- 양식의 파란색·빨간색 안내 문구는 사업별 가이드(양식매핑.md)에 원문 기록 완료 → 참고하되 최종 문서에서는 삭제
- 기술 파트는 참고자료 문서의 내용을 거의 그대로 사용
- Layer 1의 정확한 수치·명칭을 사용할 것 (임의 변경 금지)
- Layer 2의 전략 방향과 포지셔닝을 일관되게 유지할 것
- 본 파일(Layer 3)의 문장 스타일·서식 규칙을 따를 것
- 해당 사업 지시사항 가이드의 팀원 피드백·수정 원칙을 준수할 것
```

### 1-2. 작성 전 체크리스트

| 순서 | 확인 사항 |
|------|---------|
| ① | 원본 양식의 목차·섹션 구조 파악 완료 |
| ①-1 | **원본 양식 안내 문구 추출 완료** (→ 1-3 절차 참조) |
| ② | 적용할 전략 프리셋(Layer 2) 확인 (플랫폼투트랙 vs 투트랙 vs 치료제 단일) |
| ③ | Layer 1 팩트시트에서 해당 섹션에 필요한 데이터 확인 |
| ④ | 해당 사업의 심사 기준/평가 항목 확인 (있을 경우) |

### 1-3. 원본 양식 안내 문구 추출 절차

> 정부 지원사업 양식(.docx)에는 **파란색·빨간색·회색 등 색상이 있는 안내 문구**(작성 팁, 예시, 주의사항)가 포함되어 있다.
> 이 안내 문구를 사업별 가이드(`양식매핑.md`)에 원문 그대로 기록하면 작성 시 원본 양식을 왔다갔다 할 필요 없이 가이드 하나만 보고 작성할 수 있어, **누락·방향 이탈을 방지**하고 퀄리티를 높인다.

**추출 대상:**

| 구분 | 설명 | 추출 방법 |
|------|------|---------|
| 작성 목차 표 | 1페이지 목차 표의 항목별 세부 안내 | 표 내용 전체 기록 |
| 공통 작성 규칙 | 2페이지 상단 ※ 표시 규칙 (분량 제한, 양식 변경 금지, 개인정보 마스킹 등) | 전문 기록 |
| 본문 섹션별 안내 | 각 섹션 제목 아래 파란색 서술 가이드 | 섹션별 전문 기록 |
| 표 안 예시 텍스트 | 일정표·사업비·팀 구성 등 표의 파란색 샘플 데이터 | 표 형태로 기록 |
| 요약표 항목별 안내 | 개요(요약) 표 각 행의 파란색 안내 | 항목별 표로 기록 |
| 경고 문구 | 빨간색 주의사항 (개인정보 마스킹 등) | 전문 기록 |

**추출 절차:**

1. python-docx로 원본 양식 열기
2. 본문 paragraph를 순서대로 순회하면서 `run.font.color` 확인 → 파란색(0000FF, 001AFF 등) / 빨간색(FF0000) 텍스트 추출
3. 모든 table의 각 cell을 순회하면서 동일하게 색상 텍스트 추출
4. 추출된 안내 문구를 **페이지·위치별**로 사업별 가이드(`양식매핑.md`)의 "원본 양식 안내 문구 전문" 섹션에 원문 그대로 기록
5. 파트별 상세 매핑 각 섹션에서 해당 안내 문구 섹션을 참조 링크로 연결

**기록 원칙:**
- 안내 문구는 **원문 그대로** 기록 (요약·의역 금지)
- 중복 텍스트(좌우 페이지 반복)는 1회만 기록
- 색상 구분을 명시 (파란색/빨간색)
- 실제 작성 시 안내 문구는 삭제하고 검정 글씨로 우리 내용을 채움

## 2. 문장 스타일 규칙

### 2-1. 종결 어미 — 명사형 종결 우선

**핵심 원칙: ~함, ~임 같은 용언형 어미 대신, 가능한 한 명사형으로 종결한다.**

| 맥락 | ❌ 지양 (용언형) | ✅ 권장 (명사형) |
|------|---------------|---------------|
| 사실/현황 | "전신 부작용 우려가 존재함" | "전신 부작용 우려 존재" |
| 계획/예정 | "제형 최적화를 수행할 예정임" | "제형 최적화 수행 예정" |
| 판단/전망 | "타당성이 높을 것으로 판단됨" | "타당성 높을 것으로 판단" |
| 가능성 | "부작용을 최소화할 수 있음" | "부작용 최소화 가능" |
| 확인된 결과 | "동등 이상의 효과를 검증함" | "동등 이상 효과 검증 완료" |
| 보유/확보 | "IP 포트폴리오를 확보하고 있음" | "IP 포트폴리오 확보" |
| 필요성 | "추가 검증이 필요함" | "추가 검증 필요" |

**적용 기준:**
- 조사(~가, ~를, ~을) + 용언(~함, ~임, ~있음)으로 끝나는 문장 → 조사 생략 + 명사형 종결로 전환
- 단, 문맥상 명사형이 어색하거나 의미가 불명확해지는 경우에는 용언형 허용
- 긴 문장에서 주어-목적어 관계가 복잡할 때는 가독성을 위해 용언형 사용 가능

**절대 사용 금지:** 구어체(~거든요, ~잖아요), 반말(~다, ~했다), 존댓말(~합니다, ~입니다)

### 2-2. 계층 구조 (ㅇ/- 시스템)

```
ㅇ [상위 문장] 해당 섹션의 핵심 주장·결론을 1~2문장으로 요약
 - [하위 문장 1] ㅇ 문장을 뒷받침하는 근거·세부 사항
 - [하위 문장 2] 추가 근거 또는 데이터
 - [하위 문장 3] 추가 설명
```

**규칙:**
- ㅇ 문장은 굵은 핵심 키워드를 포함할 수 있음
- \- 문장은 ㅇ 문장의 세부 설명으로, 독립적으로 읽혀도 의미가 통해야 함
- ㅇ 하나에 \- 는 2~5개가 적정
- 연속된 ㅇ 없이 반드시 \- 가 따라와야 함

### 2-3. 톤 규칙

| 규칙 | 올바른 예 | 잘못된 예 |
|------|----------|----------|
| 객관적 근거 기반 | "경구 대비 1/40 수준의 투여량으로 전신 노출을 최소화하는 기술" | "혁신적인 기술" |
| 정량 우선 | "2024년 약 105억 달러 규모" | "거대한 시장" |
| 단정 회피 | "~것으로 판단됨", "~가능성이 높음" | "~이다", "~확실하다" |
| 수식어 절제 | "기존 대비 차별화된" | "최고의", "유일한", "획기적인" |

## 3. 기술 설명 작성 패턴

### 3-1. 필수 흐름 (모든 기술 설명에 적용)

```
① 기존 문제/한계 정의
    ↓
② 종래 기술의 접근 방식과 그 한계
    ↓
③ 본 기술의 해결 방식
    ↓
④ 정량적 차별성 (숫자 필수)
    ↓
⑤ 검증 결과/실적
```

### 3-2. 기전(Mechanism) 설명 구조

**"물질 → 작용 경로 → 타겟 → 결과"** 순서 유지:

> 예: "펩타이드 운반체가(물질) 세포 당단백질과의 상호작용을 바탕으로(작용 경로) 모낭 심부의 DHT 활성영역에(타겟) Finasteride를 도달시켜 치료효과를 발휘하도록 유도함(결과)"

### 3-3. 경쟁 비교 서술 구조

**"기술명 → 기전 → 장점 → 한계 → 본 기술 대비 포지션"** 순서:

> 예: "미녹시딜은(기술명) 혈관 확장을 통한 모낭 자극을(기전) 주된 기전으로 하기 때문에(장점 내포) DHT 생성 자체를 억제하지 못하며, 처치 중단 시 효과가 소실되는 단점이 있음(한계). 이들과 비교해 본 연구팀의 기술은...(본 기술 포지션)"

## 4. 시장·사업화 서술 패턴

### 4-1. 시장 현황 필수 흐름

```
① 목표시장 정의 (세그먼트)
    ↓
② 시장 규모 (절대 수치 + 출처)
    ↓
③ 성장 전망 (CAGR + 전망 연도)
    ↓
④ 세부 트렌드 (기술 축 이동, 소비 패턴 변화)
    ↓
⑤ 기회 요인 (본 기술의 시장 진입 타당성 연결)
```

### 4-2. 시장 수치 표기 규칙

- 글로벌: "2024년 약 105억 달러 규모로 추산되며, 2030년 173억 달러 수준으로 성장하여 연평균 8.7%의 성장세를 기록할 것으로 전망됨"
- 국내: "2023년 기준 약 449억 원 규모" (원화 단위)
- 그래프 캡션: 단위 명시 (예: "단위: 십억 달러")
- Layer 1의 수치를 정확히 인용할 것 (반올림/임의 변경 금지)

### 4-3. 시장 데이터 출처 표기 규칙

> ⚠️ 시장 규모·성장률·점유율 등 **구체적 수치나 통계 자료를 기재할 경우**, 반드시 인터넷 검색을 통해 원출처를 확인하고 출처를 표기할 것.

**출처 확인 절차:**
1. Layer 1에 이미 출처가 명기된 수치 → 해당 출처를 그대로 사용
2. Layer 1에 출처가 없는 수치 → 인터넷 검색으로 원출처(리서치 기관, 정부 통계, 학술자료) 확인 후 표기
3. 출처를 확인할 수 없는 수치 → 사용 지양, 또는 "업계 추정치" 등 단서 부착

**출처 표기 형식:**
- 본문 내 괄호 표기: `(출처: Grand View Research, 2024)`, `(출처: 건강보험심사평가원, 2023)`
- 표 내 표기: 표 하단 또는 비고 열에 `출처: OOO` 형태로 기재
- 복수 출처: `(출처: Data Bridge, 2024; Toward Healthcare, 2024)`

**표기 예시:**
- "2024년 약 105억 달러 규모 (출처: Grand View Research, 2024)"
- "CAGR 9.01%로 글로벌 최고 성장률 (출처: Data Bridge Market Research, 2024)"
- "국내 탈모 진료비 449억 원 (출처: 건강보험심사평가원, 2023)"

> ⚠️ 출처 불명 수치를 근거 없이 기재하면 심사위원 신뢰도 저하 요인. 출처 표기는 선택이 아닌 필수.

## 5. 서식·표기 규칙

### 5-1. 전문용어

| 유형 | 규칙 | 예시 |
|------|------|------|
| 약물명 | 영문 그대로, 첫 글자 대문자 | Finasteride, Minoxidil, Dutasteride |
| 기전명 | 한글(영문) 병기 (첫 등장 시) | 5α-환원효소 저해제(5α-reductase inhibitor) |
| 기술명 | 첫 등장 시 풀네임+약어 | Cell Penetrating Peptide(CPP) |
| 약어 | 첫 등장 시 풀네임(영문 풀네임, 약어) 형태로 표기, 이후 약어만 | 안드로겐성 탈모(androgenetic alopecia, AGA) → 이후 AGA |

### 5-1-1. 괄호 표기 규칙

| 앞 글자 | 규칙 | 올바른 예 | 잘못된 예 |
|---------|------|----------|----------|
| 한글 | 붙여 씀 | 안드로겐성 탈모(AGA) | 안드로겐성 탈모 (AGA) |
| 영문 | 한 칸 띄움 | Finasteride (1mg) | Finasteride(1mg) |
| 논문 | 학술지명 연도, 권(호), 페이지 | Biomacromolecules 2024, 25(1), 436 |
| 특허 | 출원번호 (출원일) | 10-2024-0052460 (2024-04-18) |

### 5-2. 표·그림 캡션

- **표**: `[표] 연구자가 보유한 지식재산권 현황`
- **그림**: `[그림] 연구팀 보유 기술과 기존 탈모 치료제 비교`
- 캡션은 내용을 구체적으로 명시 (대상 + 범위 포함)
- 단위가 있는 경우 캡션에 명시: `(단위: 십억 달러)`

### 5-3. 표 작성 규칙

**⚠️ 원본 양식 표 컬럼 유지 원칙:**
- 원본 양식(.docx)에 이미 존재하는 표의 **컬럼 구조(헤더, 열 수, 열 순서)는 그대로 유지**
- 표의 컬럼을 임의로 추가·삭제·병합·순서 변경하지 않을 것
- 내용(셀 값)만 채워 넣고, 행(row)은 필요에 따라 추가 가능
- 원본 양식에 표가 없는 경우에만 아래 표준 포맷을 사용

**경쟁사 비교표:**

| 구분 | 제품/기술 | 개발사 | 핵심 기전/특성 | 한계/과제 |
|------|---------|--------|-------------|---------|

**특허 현황표:**

| 번호 | 특허명 | 출원인 | 출원번호 (출원일) | 상태 |
|------|--------|--------|----------------|------|

**로드맵/일정표:**

## 7. 셀바스찬 스타일 가이드 (표·이미지 중심 서술)

> **적용 대상**: 예비창업패키지 등 10p 이내 사업계획서에서 정보 밀도를 극대화해야 하는 경우

### 7-1. 핵심 원칙

| 번호 | 규칙 | 설명 |
|------|------|------|
| 1 | **표 중심 레이아웃** | 핵심 내용은 반드시 표로 정리. 3문장 이상 산문이 연속되면 표 전환 검토. 글만 나열 금지 |
| 2 | **표 내 ○ 기호** | 비교표에서 해당 항목 표시 시 ○ 사용 |

| 4 | **캡션 표기** | 이미지·표 아래 (A), (B) 등 라벨 부여 |
| 5 | **회색 강조 박스** | ❖ 기호 + 회색 배경으로 핵심 특징·차별점 강조 |

| 7 | **여백 최소화** | 빈 공간 없이 정보 밀도 극대화 (10p 한도 내) |

### 7-2. 이미지 슬롯 표기법

- 문서 작성 시 이미지가 들어갈 위치에 아래 형식으로 표기:
  ```
  【이미지 슬롯: 펩타이드 피부 투과 모식도】
  【이미지 슬롯: 경쟁 기술 비교 도식】
  【이미지 슬롯: TRL 로드맵 타임라인】

- 이미지 삽입은 사용자가 별도 진행. 슬롯 표기는 위치와 내용 가이드 역할

### 7-3. 회색 강조 박스 예시

```
❖ 핵심 차별점
  · 이중기능 펩타이드: 피부투과 + 항염 치료 활성을 동시에 발휘하는 캐리어 프리 전달 기술
