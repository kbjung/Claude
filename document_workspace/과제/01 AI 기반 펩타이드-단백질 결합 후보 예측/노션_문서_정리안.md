## 2. [외부] 시스템 설명서 — 기록할 내용

> **목적**: 비전공자(교수님, 외부 발표)도 이해할 수 있게 핵심만 간결하게
> **톤**: 개념 중심, 수식 최소화, 다이어그램 활용

### 2-1. 파이프라인 설명서 (전체 흐름)

```
1. 시스템 개요
   - 목표: 타겟 단백질에 결합하는 펩타이드 후보 자동 도출
   - 4단계 흐름: [펩타이드 생성] → [구조 예측] → [후보군 평가] → [최종 후보 정렬]
   - 파이프라인 다이어그램 (기존 SVG 활용)

2. 파이프라인 단계별 요약
   - 각 단계: 무엇을 / 왜 / 어떤 도구로
   - 표 1장으로 정리 (단계 | 목적 | 도구 | 입력→출력)

3. 최종 평가
   - 평가 지표 3개: PRODIGY(친화도) + Vina(결합 안정성) + ipTM(구조 신뢰도)
   - PLIP: 보조 지표 (결합 패턴 참고용)
   - 정규화 후 가중 합산 → FinalScore로 후보 순위 결정

4. 구축 환경
   - 하드웨어 사양 표
   - 소프트웨어 스택 (Python, conda, WSL2)

5. 출력물
   - 최종 Excel (후보 순위표) + PDB zip (3D 구조 파일)
```

### 2-2. 파이프라인 모델 설명서 (각 모델 상세)

각 모델 페이지에 아래 구조로 통일:

```
[모델명]

■ 핵심 개념
  - 한 줄 요약
  - 비유적 설명 (비전공자용)

■ 역할
  - 파이프라인에서 무엇을 담당하는지

■ 입력 → 출력
  - 표 형식

■ 왜 이 도구를 선택했는가
  - 대안 모델 대비 장점 1-2줄

■ 신뢰도
  - 논문 정보, 인용수, 출판 저널
```

#### 각 모델별 핵심 내용

**PepMLM**
- 핵심: 타겟 단백질 서열을 보고 결합 가능한 펩타이드 서열을 AI가 생성
- 비유: "열쇠 모양을 보고 그에 맞는 자물쇠를 AI가 설계"
- 입력: 타겟 단백질 서열 → 출력: 펩타이드 후보 N개
- 선택 이유: 구조 정보 없이도 서열만으로 생성 가능, RFdiffusion 대비 히트율 우위 (38% vs 29%)
- 논문: Nature Biotechnology 2025

**ColabFold (AlphaFold-Multimer)**
- 핵심: 단백질과 펩타이드가 결합한 3D 복합체 구조를 예측
- 비유: "두 퍼즐 조각이 어떻게 맞물리는지 3D로 시뮬레이션"
- 입력: 단백질+펩타이드 서열 → 출력: 3D 구조 파일(.pdb) + 신뢰도 점수(ipTM, pTM)
- 선택 이유: AlphaFold-Multimer의 정확도 + MMseqs2의 속도
- 논문: Nature Methods 2022, Nature Protocols 2025

**OpenMM**
- 핵심: 예측된 구조의 물리적 오류(원자 충돌 등)를 보정하는 구조 안정화 단계
- 비유: "거친 조각품을 사포로 다듬어 깔끔하게 마무리"
- 입력: 복합체 PDB → 출력: 안정화된 복합체 PDB
- 선택 이유: GPU 가속 지원, AlphaFold2도 AMBER relaxation 적용
- 논문: PLoS Comput. Biol. 2017 (v7), J. Phys. Chem. B 2024 (v8)

**AutoDock Vina**
- 핵심: 단백질-펩타이드가 물리적으로 얼마나 안정적으로 결합하는지 에너지 기반 평가
- 비유: "두 조각이 얼마나 단단히 맞물리는지 물리 시뮬레이션으로 측정"
- 입력: 단백질 + 펩타이드 구조 → 출력: 결합 에너지(kcal/mol)
- 선택 이유: 가상 스크리닝 분야 표준 도구, 37,000+ 인용
- 한계: 펩타이드의 높은 유연성으로 인해 소분자 대비 정확도 제한 → ADCP 보완 검토 중
- 논문: J. Comput. Chem. 2010

**PLIP** (보조 지표)
- 핵심: 결합 부위에서 어떤 종류의 상호작용(수소결합, 소수성 등)이 일어나는지 분석
- 비유: "접착면에서 어떤 종류의 접착제가 사용되었는지 분석"
- 입력: 복합체 PDB → 출력: 상호작용 종류별 분포
- 역할 변경: 최종 점수 반영에서 제외 → 연구자 참고용 보조 지표 (2026.03.10 확정)
- 변경 이유: 정성 분석 도구로 설계됨, 정량 점수화 선행 사례 없음
- 논문: Nucleic Acids Res. 2015, 2021, 2025

**PRODIGY**
- 핵심: 복합체 구조의 접촉 특징을 분석하여 결합 친화도(ΔG, Kd)를 ML로 예측
- 비유: "결합면의 특징을 읽어서 얼마나 강하게 붙을지 예측하는 AI"
- 입력: 복합체 PDB → 출력: ΔG (kcal/mol), Kd (M)
- 선택 이유: 해석 가능한 특징 기반 모델 (White Box), PPI-Affinity(Black Box) 대비 강건성 우위
- 정확도: Pearson r = 0.73, RMSE = 1.89 kcal/mol
- 논문: Bioinformatics 2016

## 3. [내부] 파이프라인 구성 계획서 — 기록할 내용

> **목적**: 실무자(범중, 주영님)가 개발·이슈 관리에 활용
> **톤**: 코드 레벨 상세, 이슈 히스토리, 의사결정 근거 기록

### 3-1. 메인 페이지 구조

```
1. To Do List (최신 상태 유지)
   - 처리 완료: 체크 표시
   - 진행 중: 담당자, 상태 표시
   - 대기: 선행 조건 명시

2. 파이프라인 구성
   - 8단계 상세 (STEP 1~8)
   - 각 단계 입출력 파일 명시
   - 다이어그램

3. 코드 업데이트 이력
   - 고도화1: ML 기반 가중치 튜닝 (샘플: 18-mer 200개)
   - 고도화2: MSA 자동 설정, 다중 타겟 대응
   - GBSA 이슈 해결 이력
   - PLIP 보조지표 전환 (미반영 → 반영 예정)

4. 현재 진행 작업
   - ADCP 도입 검토 (주영님)
   - FinalScore 가중치 재배분 (PLIP 제외 후)

5. 이슈 관리
   - 처리 완료 이슈 (링크)
   - 미해결 이슈 (링크)
```
