#### 3. PLIP 점수 계산 방식 (상세)

**결합 종류별 중복 제한 규칙**:
- **Hydrophobic 상호작용**: 클러스터링으로 중복 제거 (같은 리간드 원자-같은 레지듀의 여러 접촉 → 가장 가까운 것만 남김)
- **수소결합 (H-bond)**: Donor는 원칙적으로 1개만 제한 (Acceptor는 다중 가능)
- **Salt bridge 존재 시**: 같은 원자쌍의 H-bond 제거

**가중치 산정**:
- 문헌의 상호작용 에너지 범위를 기준으로 선정
- 현재 평가 모델 도입이 적절한지 주영님과 논의 진행 중

**현재 상태**:
- PLIP 평가 점수는 최종 점수에 15% 반영
- 궁극적 목표: 연구자가 후보군을 평가하는 보조 지표로 활용

#### 4. 코드 업데이트 - 고도화1: 가중치 튜닝

**목표**: ML 기반 가중치 최적화 ([PRODIGY, Vina, ...] - MM-GBSA)

**선택지**:
1. **GBVI를 MOE로 계산** → 수동 처리
2. **GBVI를 Python 코드에 자동화**
3. **OpenMM 내부 MM-GBSA 활용** (추천)
   - Receptor GBSA + Ligand GBSA - Complex GBSA = MM-GBSA (Binding free energy)
   - OpenMM 내부 GBSA 계산 기능 활용

#### 5. GBSA 이슈 (해결 완료)

**문제점**:
- 초기: 양수 또는 과도하게 큰 값 (1~2천 이상)
- 원인: OpenMM이 단백질/펩타이드 분리 후 각각 minimize → 복합체 구조와 다름

**해결 방안**:
- 분리 후 minimize 없이 GBSA만 계산하도록 수정 → 대부분 정상 범위
- 추가 문제: ColabFold 생성 단계부터 문제 있는 복합체 존재
- 해결책: GBSA > 100인 복합체만 추출 → 재시도 로직 추가 (최대 3회)

**관련 문서**:
- GBSA 이상값 조사 보고서 (2차 분석)
- pepbind05.py 구현 완료 Walkthrough
- pepbind05.py 코드 (수정 완료)

#### 6. 샘플 데이터 및 평가 계획

**데이터**: 18-mer, 200개 (train:test:valid = 140:40:20)

**평가 항목**:
- 입력: 평가 모델 가중치, final_score에 따른 순위
- 평가: GBSA 값에 따른 순위

**성능 판단**:

#### 7. 고도화2 항목

**MSA 옵션 자동 설정**:
- 타겟 단백질 길이에 따라 MSA 옵션값 조정

**다중 타겟 단백질 대응**:

#### 8. 파이프라인 구성 표

| 단계 | 목적 | 도구/모델 | 비고 |
|------|------|---------|------|
| 1. 펩타이드 생성 | 생물학적으로 타당한 서열 생성 | PepMLM | 단백질 타겟 기반/무작위 |
| 2-1. 구조 예측 | 펩타이드-단백질 복합체 3D 구조 예측 | ColabFold | ipTM/pTM 신뢰도 점수 생성 |
| 2-2. 구조 튜닝 | 물리적 타당성 개선 | OpenMM | 에너지 최소화 + 짧은 relaxation |
| 3-1. 도킹 | 결합 위치/포즈 탐색 및 점수화 | AutoDock Vina | 결합 에너지(kcal/mol) 계산 |
| 3-2. 상호작용 분석 | 결합 상호작용 정량화 | PLIP | H-bond, 소수성, salt bridge 등 |
| 3-3. 친화도 예측 | 복합체 결합력 정밀 예측 | PRODIGY | ΔG, Kd 추정 |
| 4. 최종 후보 정렬 | 통합 점수 기반 선별 | 통합 점수 | ipTM + Vina + PLIP + PRODIGY |

#### 9. 각 모델 상세 설명

**PepMLM** (펩타이드 생성):
- 입력: 타겟 단백질 서열
- 출력: 펩타이드 후보 서열 리스트
- 의미: 구조 정보 없이 대량 생성

**ColabFold (AlphaFold-Multimer)** (구조 예측):
- 입력: FASTA (단백질 + 펩타이드), MSA
- 출력: PDB + 신뢰도 지표 (pLDDT, PAE, pTM, ipTM)
- 의미: 표준화된 방식의 구조 생성

**OpenMM** (구조 튜닝):
- 역할: 포스필드 포텐셜 에너지 최소화
- 포스필드 에너지 수식:
  ```
  E_total = E_bond + E_angle + E_torsion + E_vdW + E_electrostatic
  ```
  - Bond stretching (결합 길이)
  - Angle bending (결합 각도)
  - Torsion (이면각)
  - van der Waals (반데르발스)
  - Electrostatic (정전기)

**AutoDock Vina** (도킹):
- 역할: 경험적 물리 기반 스코어링으로 도킹 포즈 평가
- 기능: 포즈 탐색 + 안정성 점수화
- 출력: 상위 포즈 (.pdbqt) + affinity (kcal/mol)

**PLIP** (상호작용 분석):
- 역할: 구조 기반 결합 상호작용 정량 분석
- 출력: report.xml (H-bond, 소수성, salt bridge)

**PRODIGY** (친화도 예측):
- 역할: 인터페이스 특징 기반 친화도 예측
- 출력: ΔG, Kd

#### 10. 기타 중요 항목

**MSA 옵션 수정** (단백질 800-mer 이상 문제):
- 이전: 256:1024 → 현재: 256:512
- ColabFold 옵션: recycle=9, models=5

**펩타이드 생성 방식**:
- 기존: 무작위 생성 → 상위 10% 추출
- 개선: 상위 후보의 일부 서열 가림 → 재생성 → Multi-pass 방식
- 현재 결정: 기존대로 진행 후 구조예측에서 단백질 타겟 위치 확인 (2025.09.02 주영님과 논의 결과)

## 2️⃣ 최종 평가 방법 정리

### 문서 제목
**최종 평가 방법 정리**

### 핵심 내용

#### 1. 평가 방법 도입 로드맵

| 조합 | 구성 | 정확도 |
|------|------|--------|
| 초기 | ipTM + Vina + PLIP + Pafnucy | B급 |
| 개선1 | pTM + Vina + PLIP + PPI-Affinity | A급 |
| **현재** | **ipTM + Vina + PLIP + PRODIGY** | **A+급** |

**PRODIGY 선택 이유**:
- 2020년 이후 인용 수 급증 (연간 수십 건 → 수백 건 이상)
- 비교적 최근에 많이 사용되기 시작한 모델

#### 2. 최종 평가 점수 계산식

```
Final_Score = w1 × norm(PRODIGY) + w2 × norm(Vina) + w3 × norm(PLIP) + w4 × norm(ipTM)
```

**정규화 방식**: 0~1 범위의 고정 스케일링 (사전 정의된 최소/최대값 기준)

#### 3. 가중치 배분 (현재)

| 요소 | 가중치 | 설명 |
|------|--------|------|
| **PRODIGY Score** | **50%** | 결합 친화도를 가장 직접적으로 예측 |
| **Vina Score** | **25%** | 물리 법칙 기반 결합 안정성 (PRODIGY 보조) |
| **PLIP (Total Interactions)** | **15%** | 물리적 상호작용 정량화 (보조 지표) |
| **ipTM Score** | **10%** | 구조 예측 신뢰도 (기본 전제) |

#### 4. 가중치 설정 근거

**PRODIGY (50%)** - 최고 가중치
- 파이프라인의 최종 목표: 결합 친화도 높은 펩타이드 찾기
- PRODIGY: 이 값을 가장 직접적으로 예측하는 모델
- 논리: 핵심 지표에 최고 가중치 부여는 당연

**Vina (25%)** - 2번째 높은 가중치
- 물리 법칙 기반 결합 안정성 지표
- PRODIGY의 핵심적 보조 점수
- 적절한 비중

**PLIP (15%)** - 보조 가중치
- 상호작용 '개수'는 결합 '질'을 완벽히 대변하지 못함
- 몇 개의 중요한 상호작용 > 많은 약한 상호작용
- 결합 타당성 뒷받침 역할

**ipTM (10%)** - 가장 낮은 가중치
- pTM = 결합력 아님, "입력 구조의 신뢰도"
- 점수 낮으면 모든 평가가 무의미해지는 '기본 전제'
- 필수적이나 비중 낮음

#### 5. 최적 가중치 찾는 방법 (과학적 접근)

**목표**: Final_Score와 실제 실험값 간 상관계수 최대화

**방법 1: Grid Search (격자 탐색)**
1. 벤치마크 데이터셋 준비 (예: PDBbind)
2. 각 평가 모델 점수 사전 계산
3. 모든 가중치 조합 테스트 (w1+w2+w3+w4=1.0)
4. 각 조합의 상관계수 계산
5. 가장 높은 상관계수의 가중치 조합 선택

**제안**:
- **옵션 1 (좋음)**: 현재 가중치 그대로 사용
  - 이미 논리적으로 충분히 타당
  - 파이프라인 성능 잘 대표 가능성 높음

- **옵션 2 (더 좋음)**: 벤치마크 테스트로 최적 가중치 찾기
  - 객관적 근거 확보
  - "우리 파이프라인 상관계수 0.85, 최적 가중치는..." 등으로 신뢰도 주장 가능

#### 6. 최종 후보 선정

상위 N개 구조를 최종 후보로 선정 (상세 방식은 문서에 미기재)

### 특이사항

1. **⚠️ PLIP 제외 논의 진행 중 (메인 문서와 충돌)**
   - 메인 문서 CLAUDE.md에서: PLIP 제외 확정 (2026.03.10 교수님 승인)
   - 이 문서: PLIP 15% 가중치로 포함
   - **결론**: 최종 평가식이 코드에 반영되지 않음 (아직 수정 대기)

2. **현재 상태**: 이 문서는 A+급 평가 방식을 설명하지만, 최종 결정은 PLIP 제외로 변경됨

## 3️⃣ 중간 산출물 설명서

### 문서 제목
**중간 산출물 설명서**

### 핵심 내용

#### 1. ColabFold 출력: PDB 파일

**예시 파일명**:
```
complex_0_unrelaxed_rank_001_alphafold2_multimer_v3_model_1_seed_000.pdb
```

**의미**:
- 단백질 + 펩타이드 복합체의 3D 구조 좌표 파일
- 각 원자의 정보 포함:
  - 원자명 (N, CA, C, O 등)
  - 아미노산 이름
  - 좌표 (x, y, z)
  - 체인 ID

**파이프라인 역할**:
- 구조 기반 후속 모델(Vina, PLIP, PRODIGY)의 기초 토대
- ipTM/pTM 신뢰도 평가 근거

**주의사항**:
- "실제 물리적 구조의 추정치"
- 구조 이상 → 후속 모든 평가 흔들림

#### 2. Vina 입력용 구조: PDBQT

**의미**:
- PDB + charge + atom type + torsion 정보 추가
- Vina 전용 구조 파일

**구성**:
- receptor.pdbqt → 단백질 (대체로 rigid)
- ligand.pdbqt → 펩타이드 (토션 회전 표시 + 전하 정보)

**필요 이유**:
- Vina의 scoring function이 필요로 하는 정보:
  - 전하 (charge)
  - 원자 타입 (atom type)
  - 회전 가능한 결합 (rotatable bonds)
- PDB에는 이 정보 없음

**운영자 주의사항**:
- "PDBQT는 Vina가 이해할 수 있는 형태로 변환한 PDB"
- Convert 실패 시 도킹 오류 발생 가능
- 체인 분리, 수소 추가 여부, charge 계산 단계에서 오류 가능성

#### 3. Vina 출력: 도킹 포즈 + 로그

**의미**:
- Vina가 찾은 수많은 포즈 중 상위 N개 포즈의 좌표
- 각 포즈별 결합 점수(ΔG), RMSD 포함

**파이프라인 역할**:
- 에너지 기반 재정렬(Scoring)의 핵심 근거

**운영자 주의사항**:
- "Vina score는 구조 생성 + 내부 에너지 계산 결과"
- 포즈가 이상하면 PLIP, PRODIGY도 모두 영향

#### 4. PLIP 출력: report.xml

**의미**:
- 구조 기반 상호작용 분석 결과

#### 이슈 1: PepMLM 펩타이드 생성 방식의 문제

**문제 인식**:
- PepMLM이 "타겟 단백질과의 결합 확률"을 학습한 모델이 아님
- 실제로는: 타겟 단백질의 한쪽 끝에서 연장 시 "다음에 나올 서열"을 확률적으로 산출하는 모델
- 현재 생성 펩타이드: 설정 길이의 가능한 서열을 나열한 것

**근거**:
- PepMLM 학습 방식: '[단백질 서열] [펩타이드 서열]' 형태
- 단백질 중간에 붙은 펩타이드도 동일한 형식으로 학습

**개선 필요**:
- 원하는 결과 도출을 위해: '[타겟 단백질 서열] [mask] [mask] ...' 형식 필요

**보안 방법** (현재 진행 중):
1. 기존 방식으로 후보군 산출 → 상위 10% 추출
2. 상위 후보 서열의 일부 가림 → 재생성
3. 여러 번 반복 → 각 단계 상위 10% 모아 FinalScore 비교
4. **이점**: 기존 높은 점수 후보를 기반으로 더 높은 점수 후보 기대

**최적화 질문**:
- ColabFold/평가 모델에서 최종 점수에 가장 영향 주는 서열 위치를 알 수 있는가?

- 알 수 없다면: 처음/끝 고정 또는 전체 재생성

## 8️⃣ PPI-Affinity vs PRODIGY

### 문서 제목
**PPI-Affinity vs PRODIGY**

### 핵심 내용

#### 1. 두 모델의 근본적 차이

##### PPI-Affinity (패턴 기반 ML / Black Box)

**작동 원리**:
1. 3D 구조 전체를 복잡한 **패턴(그래프)**으로 인식
2. AI 모델(GNN/SVM)이 데이터 내 패턴을 **스스로 학습**
3. 학습된 패턴으로 결합력 예측

**기술 구조**:
- **입력**: 단백질-펩타이드 3D 구조
- **변환**: 그래프(Graph) 형태로 변환
  - 각 원자/잔기 = 노드 (vertex)
  - 상호작용(거리, 결합) = 엣지 (edge)
- **모델**: GNN (그래프 신경망) 또는 SVM (서포트 벡터 머신)
- **학습**: 훈련 데이터의 결합 패턴을 통계적으로 학습
- **출력**: 친화도 예측 점수 (Black Box)

**장점**:
- 데이터가 충분하면 **복잡한 상호작용 패턴** 포착 가능
- 예: 수소결합, π-π stacking, van der Waals, 복합 상호작용

**단점**:
- 예측 근거가 상대적으로 **불투명**
- "왜 특정 친화도를 예측했는가" 해석 어려움
- 훈련 데이터에 **없는 새로운 구조적 패턴에 취약**
- **데이터 의존성** 높음

##### PRODIGY (특징 기반 ML / White Box)

**작동 원리**:
1. 3D 구조에서 **물리적으로 의미 명확한 특징** 직접 계산
2. 계산된 특징들을 기반으로 결합력 예측
3. 예측 근거가 **명확하고 해석 가능**

**기술 구조**:
- **입력**: 단백질-펩타이드 3D 구조 (PDB)
- **특징 추출**: 물리적으로 해석 가능한 특징들 직접 계산
  - 결합 부위의 원자 접촉 수 (contact count)
  - 극성/비극성 잔기 수
  - 비극성 표면적 (hydrophobic surface area)
  - 소수성 상호작용 정도
  - 기타 물리적 특징들
- **모델**: 회귀 모델 (Random Forest, Linear Regression 등)
- **학습**: 계산된 특징과 친화도(Kd, ΔG)의 상관관계 학습
- **출력**: 친화도 예측 점수 (물리적 근거 제공)

**장점**:
- **해석 가능성** 높음
  - "왜 강하게 결합하는가" 설명 가능
  - 예: 비극성 표면적 ↑ → 소수성 효과로 결합 안정성 ↑
- **일관성** 높음
- **강건성** 높음 (새로운 구조에도 적용 가능)

**단점**:
- 미리 정의되지 않은 **새로운 패턴은 반영 불가**
- 특징 설계의 정확성이 중요

#### 2. 단계별 계산 방식 비교

| 단계 | PPI-Affinity | PRODIGY |
|------|-------------|---------|
| **입력** | 복합체 3D 구조 (PDB) | 복합체 3D 구조 (PDB) |
| **처리 방식** | 그래프로 변환 → GNN/SVM 입력 | 구조 특징(접촉 수, 표면적 등) 직접 계산 |
| **학습** | 데이터 내 상호작용 패턴을 통계적으로 학습 | 계산된 특징과 친화도(Kd, ΔG)의 상관관계 학습 |
| **출력** | 친화도 예측 점수 (Black Box) | 친화도 예측 점수 (물리적 근거 제공) |
| **해석력** | 낮음 (패턴 기반) | 높음 (물리적 특징 기반) |
| **강건성** | 낮음 (데이터 의존적) | 높음 (물리적 특징 기반) |

#### 3. PRODIGY가 더 적절한 이유

**이유 1: 평가 방식의 일관성 (Consistency)**
- Vina, PLIP과 동일하게 3D 구조로부터 **물리적으로 의미 있는 값** 계산
- 동일한 철학: 구조 기반 물리 평가

**이유 2: 해석력**
- 결합 부위의 특징(원자 수, 표면적 등)이 명확히 정의됨
- 연구자가 "왜 이 펩타이드가 좋은가"를 설명 가능

**이유 3: 강건성**
- 물리적 특징 기반 → 새로운 데이터/구조에도 일반화 가능
- 훈련 데이터 범위 의존성 낮음

**이유 4: 투명성**
- 특징 계산 과정 공개 가능
- 재현성 높음

### 특이사항

1. **비교 분석의 명확성**
   - 각 모델의 작동 원리를 그래프, 표 등으로 상세 설명
   - 장단점을 객관적으로 제시

2. **철학적 차이의 강조**
   - "Black Box vs White Box"
   - "패턴 학습 vs 물리적 특징"
   - 이것이 PRODIGY 선택의 핵심 이유

3. **일관성의 중요성**
   - 파이프라인 전체가 "물리적 의미 기반 평가" 철학을 유지
   - PRODIGY는 이 철학과 일치
