중간 산출물 설명서

✔ 중간 산출물 최소 이해 버전

1. ColabFold 출력: PDB (Protein Data Bank format)

예: complex_0_unrelaxed_rank_001_alphafold2_multimer_v3_model_1_seed_000.pdb

이것이 의미하는 것 (개념)

  • 단백질 + 펩타이드 복합체의 3D 구조를 담고 있는 “좌표 파일”
  • 각 원자의
    • 원자명 (N, CA, C, O 등)
    • 아미노산 이름
    • 좌표 (x, y, z)
    • 체인 ID

      등이 적혀 있음.

파이프라인에서의 역할

  • 구조 기반 후속 모델(Vina, PLIP, PRODIGY)의 기초 토대
  • ipTM/pTM 신뢰도는 이 구조에 대한 평가

운영자가 알아야 할 최소 개념

  • “이 파일은 실제 물리적 구조의 추정치”이며
  • 구조가 이상하면 후속 모든 평가가 흔들린다.

2. Vina 입력용 구조: PDBQT

이것이 의미하는 것 (개념)

PDB + charge(전하) + atom type + torsion 정보가 추가된 Vina 전용 구조 파일.

  • receptor.pdbqt → 단백질 (대체로 rigid)
  • ligand.pdbqt → 펩타이드 (토션 회전 가능한 부분 표시 + 전하 정보 포함)

왜 필요한가?

  • Vina의 scoring function은 전하, atom type, rotatable bonds 정보를 필요로 하고
  • PDB에는 이런 정보가 없어서 PDBQT로 변환해야 함.

운영자가 알아야 할 최소 개념

  • “PDBQT는 Vina가 이해할 수 있는 형태로 변환한 PDB”
  • 내부 내용까지 파고들 필요 없음
  • 다만 convert 실패 시 도킹 오류가 발생할 수 있으므로
    • 체인 분리
    • 수소 추가 여부
    • charge 계산

      같은 단계에서 오류가 날 수 있음 정도만 알고 있으면 충분함.

3. Vina 출력: 도킹 포즈(.pdbqt) + 로그(stdout)

이것이 의미하는 것 (개념)

  • Vina가 찾은 수많은 포즈 중 가장 에너지 좋은 상위 N개 포즈의 좌표
  • 각 포즈별
    • 결합 점수(ΔG)
    • RMSD

      등을 포함.

파이프라인에서의 역할

  • 에너지 기반 재정렬(Scoring)의 핵심 근거

운영자가 알아야 할 최소 개념

  • “Vina score는 구조 생성 + 내부 에너지 계산 결과”
  • 포즈가 너무 이상하면 PLIP, PRODIGY도 전부 영향 받음
  • 필요하면 refined pose를 PDB로 변환해 후속 분석에 사용 가능

4. PLIP 출력: report.xml

이것이 의미하는 것 (개념)

  • 구조 기반 상호작용 분석 결과
  • 수소결합, 소수성, salt bridge 등 결합 패턴 정보

운영자가 알아야 할 최소 개념

  • “PLIP는 구조를 기준으로 결합을 판정한다 → 구조가 바뀌면 결과도 바뀐다.”

5. PRODIGY 출력: ΔG, Kd

이것이 의미하는 것 (개념)

  • 인터페이스 특징(접촉 수, 극성 잔기 비율 등) 기반 친화도 예측
  • 정적 평가 → 구조가 바뀌면 결과도 당연히 바뀜

✔ 결론: 어떤 수준으로 이해하면 가장 효율적일까?

너의 목표가 “AI 기반 펩타이드–단백질 결합 예측 플랫폼 구축”이라면 아래 수준이 가장 효율적임:

★ 반드시 이해해야 하는 것

  1. 각 모델의 역할
    • ColabFold = 구조 생성
    • Vina = 포즈 탐색 + 에너지 평가
    • PLIP = 구조 기반 상호작용 분석
    • PRODIGY = 구조 기반 ΔG 추정
    • ipTM = ColabFold 구조 신뢰도
  1. 각 산출물이 다음 단계에 어떤 영향을 주는지
    • 구조가 이상하면 후속 모든 지표가 흔들림
    • Vina 포즈를 썼다면 PLIP/PRODIGY도 해당 포즈 기준으로 다시 계산해야 함
  1. 문제가 생기는 지점
    • ColabFold output 이상 → 전 단계 스톱
    • PDB→PDBQT 변환 오류 → Vina 실패
    • PLIP/PRODIGY 입력 구조 문제 → 결합 패턴 잘못 나옴

★ 굳이 깊게 알 필요 없는 것

  • PDB 내부 원자 번호 규칙
  • PDBQT 토션 플래그 설정 규칙
  • Vina의 세부 scoring function 공식
  • XML 내부 세부 태그 구조 전체

즉, “구조 파일의 역할과 정보 흐름”만 이해하면 충분하고,

파일 포맷의 세부 사양까지 알 필요는 없음.

✔ 요청한 대로, 중간 산출물 개념 요약

아래 문장 그대로 문서에 넣어도 됨:

  • PDB: 단백질·펩타이드의 3D 구조(좌표)를 담은 기본 포맷으로, 구조 기반 분석 모델들이 공통적으로 사용하는 입력.
  • PDBQT: Vina 전용 구조 포맷으로, PDB에 전하·atom type·회전 가능한 결합 정보가 추가된 형태. 도킹·스코어링에 필수.
  • Vina pose(.pdbqt): Vina가 탐색한 최적 결합 포즈의 좌표로, 에너지 기반 재순위화에 사용됨.
  • PLIP report.xml: 구조 내 상호작용 패턴(수소결합, 소수성, salt-bridge 등)을 정량화한 분석 결과.
  • PRODIGY ΔG/Kd: 인터페이스 특징 기반 결합 친화도 예측값으로, 정적 구조 평가에 사용됨.
