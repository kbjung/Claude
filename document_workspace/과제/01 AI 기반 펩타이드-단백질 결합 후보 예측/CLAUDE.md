## 1. 프로젝트 개요

- **제목**: AI 기반 단백질 결합 펩타이드 후보 예측 시스템
- **목표**: 타겟 단백질에 특이적으로 결합하는 펩타이드 후보군을 AI 파이프라인으로 자동 도출
- **담당자**: 김범중 (대학원생), 주영님 (박사 과정), 김동혁 교수님 (지도교수)
- **코드**: `pepbind06.py` (pepbind05.py 기반, Vina score_only 적용. conda env: `pepbind_openmm`, Python 3.11)
- **실행 환경**: i7-12700 / RTX 3070 8GB / 32GB RAM / WSL2 Ubuntu 22.04.5 LTS

---

## 2. 파이프라인 구성 (현재 코드 기준)

```
[타겟 서열] → PepMLM → ColabFold → OpenMM → Vina → PLIP → PRODIGY → 재시도 → 최종 출력
```

| 단계 | 도구/모델 | 역할 | 비고 |
|------|---------|------|------|
| STEP 1 | - | 입력/경로 설정 | 타겟 서열, 펩타이드 길이, 생성 개수 |
| STEP 2 | **PepMLM** (ESM-2) | 펩타이드 후보 서열 생성 | GPU 사용 |
| STEP 3 | **ColabFold** (AF-Multimer) | 복합체 3D 구조 예측 | ipTM/pTM 신뢰도 점수 생성 |
| STEP 3b | **OpenMM** | 구조 튜닝 (에너지 최소화 + short MD) | GPU → CPU 폴백 |
| STEP 4 | **AutoDock Vina** | 도킹 (결합 안정성 kcal/mol) | CPU |
| STEP 5 | **PLIP** | 상호작용 분석 | ⚠️ 보조 지표로 변경됨 (아래 참조) |
| STEP 6 | **PRODIGY** | 친화도 예측 (ΔG, Kd) | ML 기반 |
| STEP 7 | - | 실패 복합체 재시도 | GBSA>100 시 ColabFold부터 재실행, 최대 3회 |
| STEP 8 | - | 최종 Excel + PDB zip 출력 | FinalScore 랭킹 |

## 3. 최종 평가 점수 (FinalScore)

### 현재 코드 상 수식 (pepbind05.py)
```
FinalScore = 0.50 × norm(PRODIGY) + 0.25 × norm(Vina) + 0.15 × norm(PLIP) + 0.10 × norm(ipTM)
```

### ⚠️ 변경 확정 사항 (2026.03.10 교수님 승인)
- **PLIP 점수를 FinalScore에서 제외** → 보조 지표로만 사용
- 근거: PLIP 분석 보고서 (`03 파이프라인 모델 설명서/보고서/PLIP_분석_보고서_20260304_1646.docx`)
  - PLIP은 결합 패턴 탐지(정성 도구)로 설계됨, 정량 점수화 선행 사례 없음
  - 소수성 결합 클러스터링으로 면적 정보 손실 → 점수화 부적합
  - 주영님 논의 완료 + 교수님 승인 완료
- **코드 수정은 아직 미반영** → PLIP 제외 후 가중치 재배분 필요
  - 예상 후보: PRODIGY / Vina / ipTM 3개 지표로 재배분, 또는 ADCP 도입 후 결정

## 4. 현재 진행 중인 작업

### ADCP (AutoDock CrankPep) 도입 검토
- **제안자**: 주영님
- **배경**: AutoDock의 펩타이드 전용 도킹 도구
- **목적**: 현재 Vina의 펩타이드 도킹 한계 보완 가능성 탐색
- **진행 방법**:
  1. ADCP 내용 조사 (진행 중)
  2. pepbind05.py 코드 수정하여 ADCP 값 산출
  3. MOE 프로그램의 GBVI 값과 비교
  4. 상관성 확인 → 쓸만한지 평가
- **상태**: 조사 단계
