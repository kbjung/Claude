## 1. 파일명 규칙 (공통)

기본 형식:
```
[용도]_[프로젝트/주제]_[날짜YYYYMMDD]_[시간HHMM].[확장자]
```

- 예시: `랩미팅_AI펩타이드시스템_20260323_1754.pptx`
- 예시: `PLIP_분석_보고서_20260304_1646.docx`
- 예시: `시스템_설명서_구성안_20260409_1218.md`

> 09_ai 뉴스 파일 등 도메인별 변형 규칙은 해당 폴더의 CLAUDE.md를 따른다.

## 2. 클로드 작업 시 필수 규칙

- **구성안 등 반복 수정 문서**: 업데이트 시마다 새 파일로 저장 (덮어쓰기 금지)
- **날짜/시간**: 한국 시간(KST) 기준 `YYYYMMDD_HHMM` 형식 (반드시 `TZ='Asia/Seoul' date` 명령으로 실제 KST 확인 후 사용)
- 새 파일 첫 줄에 이전 버전 파일명 기록
- **파일 생성·수정 완료 시**: 응답 마지막에 반드시 `computer://` 링크를 제공하여 사용자가 바로 열어볼 수 있도록 한다

---

## 3. HTML 페이지 작성 시 CSS 체크리스트 (뉴스/리포트 등)

표·카드형 페이지에서 한국어 태그/배지가 잘리는 이슈가 반복 발생. 신규 HTML 작성 시 아래 체크 필수:

- **`.tag` 류 인라인 배지**: 반드시 `display: inline-block;` + `white-space: nowrap;` 지정 (한국어 4자 이상 텍스트도 줄바꿈/잘림 없이 표시)
- **테이블 분류 칼럼**: `td`에 `white-space: nowrap; width: 1%;` 적용 — 분류·상태 칼럼이 콘텐츠 너비만큼만 차지하고, 본문 칼럼이 남은 공간 사용
- **카드 내부 메타 영역**: 여러 링크/구분점이 들어갈 경우 `flex-wrap: wrap;` 추가 (모바일에서 줄바꿈 보장)
- **반응형 분기점**: `@media (max-width: 768px)`에서 grid 컬럼 1열, stats-row 2열 등 기존 패턴 유지
- 작성 완료 후 한국어 6자 이상 태그(예: "데이터베이스", "파트너십", "임상 데이터")가 포함된 셀이 있다면 시각적으로 한 번 검수
