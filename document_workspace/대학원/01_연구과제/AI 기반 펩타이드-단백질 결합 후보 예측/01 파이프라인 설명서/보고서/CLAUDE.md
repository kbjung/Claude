## 6. 보고서 생성 방식 (기술 메모)

### 6-1. 도구

- **docx-js** (npm package `docx`) 사용. 전역이 아닌 `outputs/build/` 내 로컬 설치
- 생성 스크립트: `outputs/build/generate_v*.js`
- 한 번에 1,000줄+ 큰 파일을 다루므로 **Write 도구가 파일 끝을 잘라 NULL 바이트로 채우는 현상**이 발생함 — 반드시 `node -c` 또는 `wc -l`로 끝까지 정상인지 확인 필요
- 잘림 발생 시 복구: `head -n N` + 결락 부분 별도 Write + `cat`으로 합치기

### 6-2. 헬퍼 함수 (generate_v*.js 공통)

| 함수 | 용도 |
|------|------|
| `h1, h2, h3` | 헤딩 (1F4E79, 2E75B6, 404040 색상) |
| `p` | 일반 단락 |
| `bullet` | 불릿 (한국어 `•`, 들여쓰기 자동) |
| `easyBox(title, lines[])` | 회색 콜아웃 박스 (💡 이모지 + 회색 배경 #EEEEEE) |
| `quoteBox(text, source)` | 좌측 파란 보더 인용 박스 |
| `makeTable(headers, rows, columnWidths)` | 표 (DXA 단위, 헤더 배경 #1F4E79, 흰 글씨, 짝수 행 #F2F2F2 stripe) |
| `modelSubHead(letter, title)` | 모델 챕터 A/B/C/D/E/F 서브헤딩 |

### 6-3. 검증

```bash
python3 "skills/docx/scripts/office/validate.py" output.docx
```

### 6-4. PDF 미리보기 시 한글 깨짐

LibreOffice가 한글 폰트를 가지지 않아 PDF 변환 시 박스로 보임. 실제 docx에는 한글 정상 저장됨(pandoc으로 확인 가능). 사용자 PC(Windows + 맑은 고딕)에서는 정상 표시.

## 7. 누적 사용자 요청 변경 이력 (대화 흐름)

### 7-1. v2 (분야 동향 + 시스템 위치 추가)
1. 새 파일로 분리
2. 분야 동향(3장)은 시스템 개요 뒤 위치
3. 회색 콜아웃 박스로 "쉬운 설명" 표현
4. 분량 많아도 6단 구조 유지
5. 참고문헌 [n] 번호 인용
6. 문서 형식: Word(.docx) 단일 (PPT는 추후 별도 작성)

### 7-2. v2.1 → v2.2 (8개 항목 → 4개 항목)
- 6개 패러다임 표 유지
- `+`, `—` 기호 자연어 대체
- "리뷰[14]" → "리뷰논문[14]"
- 모델별 학습 데이터 항목 한눈에 표에 통일
- PepMLM 인용수 추가, PRODIGY 원논문 인용수 명시
- Vina 가중치 결정 배경 추가
- MM-GBSA: 단기 도입 후보 → 중장기 (환경 제약 반영, RTX 3070 8GB 한계, Colab Pro+ 시도 실패 이력)
- 13.4 절 제거 (사용자 본인 확인용)
- 데이터셋 상세화 (정확한 명칭, 규모, 출처)
- 인용수에 확인 날짜 명시

### 7-3. v2.3 (학습 절차·정답값·ML 로직 상세)
- 각 모델의 D 섹션에 학습 사이클 단계별 절차 추가 (입력 → 예측 → 정답 비교 → 가중치 조정)
- 정답값(Ground Truth), 손실함수, 최적화 방법 명시
- PepMLM D-2 절에 "머신러닝 핵심 개념" 박스 (전체 공통 참고 자료)
- PepMLM D-3 절에 ESM-2 사전학습 + PepMLM 파인튜닝 단계별 절차
- ColabFold D-2: AF2 학습(FAPE 손실, 170,000 단량체, 14,000 멀티머 등)
- OpenMM D-2: 포스필드 파라미터 fitting (양자역학 정답값)
- Vina D-2-1: 학습 사이클 7단계 + 구체 수치 예시
- PRODIGY D-2: OLS 닫힌 해, Leave-one-out cross-validation
- PLIP D-2: 임계값 결정 과정 (규칙 기반도 결정학·양자역학 정답 사용)
