2. 시스템 구성

2-1. 파이프라인 개요

2-2. 파이프라인 구성

  • 파이프라인 단계별 구성표
단계목적도구 / 모델비고
1. 펩타이드 생성생물학적으로 타당한 펩타이드 서열 생성PepMLM단백질 타겟 기반 또는 무작위 생성을 지원
2-1. 구조 예측펩타이드–단백질 복합체 구조 예측ColabFold (AlphaFold-Multimer)ipTM / pTM(구조 신뢰도 지표) 점수와 구조 파일(.pdb) 생성
2-2. 구조 튜닝예측 구조의 물리적 타당성 개선OpenMM에너지 최소화 및 짧은 relaxation
3-1. 도킹결합 위치/포즈 탐색 및 점수화AutoDock Vina구조기반 결합 에너지(kcal/mol) 계산

3-3. 친화도 예측복합체 결합력 정밀 예측PRODIGY구조 기반 ML로 ΔG(kcal/mol)와 Kd 추정
4. 최종 후보 정렬통합 점수 기반 최적 후보 선별ipTM + Vina + PLIP + PRODIGY각 점수 정규화 후 가중 평균 또는 다기준 필터링
  • 파이프라인 다이어그램

3. 파이프 라인 구성 모델

1. 펩타이드 생성

  • 도구/모델: PepMLM
  • 역할: 타켓 단백질에 결합 가능성이 있는 선형 펩타이드 후보 서열 생성
  • 입력: 타겟 단백질 정보(구성 서열 문자열)
  • 출력: 펩타이드 후보 서열 리스트
  • 파이프라인에서의 의미: 구조 정보 없이도 후보 서열 대량 생성

2-1. 구조 예측

  • 도구/모델: ColabFold(AlphaFold-multimer)
  • 역할: 단백질–펩타이드 복합체의 3D 구조 예측 및 예측 신뢰도 점수 제공
  • 입력: .fasta(단백질 + 펩타이드), MSA(다중서열정렬)
  • 출력: 복찹체 구조 파일(.pdb) + 신뢰도 지표(pLDDT, PAE, pTM, ipTM 등)
  • 파이프라인에서의 의미: PepMLM이 만든 후보를 “구조적으로 가능한 결합 형태”로 구체화하고, 이후 도킹/상호작용/친화도 평가 단계의 입력 구조를 표준화된 방식으로 생성

2-2. 구조 튜닝

  • 도구/모델: OpenMM
  • 역할: 물리 기반 포스필드 포텐셜 에너지가 최소가 되도록 복합체의 원자 좌표를 수치적으로 최적화하는 구조 안정화 단계
  • 입력: 복합체 구조 파일(.pdb)
  • 출력: 튜닝된 복합체 구조 파일(.pdb) + 튜닝 로그(에너지 변화, 수렴 여부 등)
  • 파이프라인에서의 의미: “구조 평가/친화도 계산이 가능한 수준”으로 좌표를 정제해, 실패율(에러/폭주 에너지/비정상 GBSA)을 낮추는 전처리 단계
  • 설명
    • 포스필드는 원자 간 상호작용을 고전역학 함수로 표현한 잠재에너지 모델
    • OpenMM은 이 포스필드 에너지가 최소가 되도록 좌표를 수치적으로 최적화
    • 이는 구조 안정화 과정이지, 직접적인 결합 자유에너지 계산은 아님
  • 추가 설명
    • 포스필드 에너지(고전역학 기반 내부 포텐셜 에너지)
      • 결합 길이 (Bond stretching)
      • 결합 각도 (Angle bending)
      • 이면각 (Torsion)
      • 반데르발스 (van der Waals)
      • 정전기 (Electrostatic)

3-1. 도킹

  • 도구/모델: AutoDock Vina
  • 역할: 경험적 물리 기반 스코어링 함수를 사용하여 탐색된 도킹 포즈 중 최소 스코어를 갖는 포즈의 결합 안정성 점수(kcal/mol)를 산출하는 평가 단계
    • → 자세(포즈) 찾기 + 안정성 점수 / 어떤 포즈가 가장 안정적인가?
  • 입력: 리셉터(단백질) pdbqt + 리간드(펩타이드) pdbqt + 탐색 박스(center/size) 및 옵션(conf)
  • 출력: (기본 도킹 모드) 상위 포즈 .pdbqt + 로그(log) 내 affinity(kcal/mol) / (스코어만) 점수(log)
  • 파이프라인에서의 의미: ColabFold(AlphaFold-Multimer)가 만든 복합체 구조를 “물리 기반 점수”로 한 번 더 현실 체크하는 단계. AI 신뢰도(ipTM/pTM)와 별개로, 결합 안정성의 근사치를 빠르게 제공해 최종 랭킹의 설득력을 높임
  • 설명
    • 3D 구조를 기반으로 리간드–단백질 결합 포즈를 탐색하고, 경험적 스코어링 함수로 결합 안정성을 추정하는 분자 도킹 프로그램
    • 포즈 탐색과 스코어링을 동시에 진행(모델 설계상 스코링만 계산은 불가)
    • 가장 낮은 에너지 포즈를 상위 후보로 제시
    • 경험적 스코어링 함수로 계산된 근사적 결합 안정성 점수
    • 경험적 스코어링 함수의 항
      • Steric (van der Waals)
      • Hydrogen bonding
      • Hydrophobic interactions
      • Conformational penalty
      • Rotational entropy penalty
    • 출력되는 점수는 입력 포즈의 에너지가 아니라, 탐색 과정에서 찾은 최소 에너지 포즈의 스코어

3-2. 상호작용 분석

  • 도구/모델: PLIP
  • 역할: 구조예측으로 얻은 복합체 구조에서 결합의 “근거(어떤 잔기-어떤 상호작용)”를 구조적으로 설명 가능한 형태로 뽑아줌
  • 입력: 복합체 구조 구조 (.pdb)
  • 출력: PLIP 리포트 파일(.xml)
  • 파이프라인에서의 의미:
    • Vina 점수 같은 “숫자 1개”를 “설명 가능한 근거”로 분해(예: H-bond 몇 개, salt bridge 어디, pi-stacking 위치 등)
    • 잘못된 포즈(현실적으로 상호작용이 거의 없거나 clash성 접촉만 있는 포즈)를 빠르게 걸러내는 1차 구조 필터로 활용
    • 최종 후보 보고서에서 결합 근거(핫스팟 잔기) 제시용
  • 설명
    • 현재 가중치는 PLIP 공식 모델이 아니라, 도킹 후보 간 상대 랭킹을 위한 휴리스틱 파라미터
    • 물리적 상호작용 강도(ionic > H-bond > hydrophobic)라는 일반적인 결합 에너지 기여도를 반영하여 설정
    • “한 원소(원자/잔기)에 여러 결합이 걸리는 경우” PLIP에서 개수 세는 방식(참고: 깃허브 문서)
      1. PLIP의 기본 단위는 “상호작용 1건 = 특정 타입의 기하학 조건을 만족하는 원자(또는 원자군) 쌍/구조쌍” 입니다
        • 즉, 같은 잔기(residue)나 같은 원자(atom)가 여러 상호작용에 동시에 관여할 수 있습니다(타입/상대 원자/기하가 다르면 별도 1건).
      1. 다만, 타입별로 “중복 폭발”을 막기 위한 필터/제약이 명시돼 있습니다 (PLIP 문서에 룰이 적혀 있음)
        • Hydrophobic contacts
          • 2단계 클러스터링으로 개수를 “대표 접촉만” 남기게 줄입니다
            • 같은 ligand atom이 같은 residue의 여러 단백질 원자와 접촉하면 가장 가까운 것 1개만 유지
            • 반대로 같은 protein atom이 여러 ligand 원자와 접촉하면 가장 가까운 것 1개만 유지
        • Hydrogen bonds
          • donor는 원칙적으로 1개 H-bond만 허용(여러 후보면 180°에 더 가까운 각도를 우선으로 1개만 유지)
          • acceptor는 여러 H-bond를 가질 수 있음(bifurcated hydrogen bonds 허용)
        • Water bridges
          • “물 1분자가 ligand-단백질 사이를 잇는 케이스”를 보고, 물이 donor로 참여하는 H-bond는 최대 2개로 제한(후보가 더 많으면 각도 기준으로 2개만 유지)
        • Salt bridge vs H-bond 중복 제거
          • 같은 그룹이 salt bridge를 이미 형성하면 그 원자들 사이의 H-bond는 제거(둘 다 세지 않음)

3-3. 친화도 예측

  • 도구/모델: PRODIGY
  • 역할: 복합체 구조에서 계산된 인터페이스 접촉 특징을 기반으로 회귀 모델을 통해 결합 자유에너지(ΔG)와 해리상수(Kd)를 예측하는 구조 기반 친화도 평가 단계
    • → 그 자세의 실제 결합 세기 예측 / 이 포즈는 얼마나 강하게 결합하는가?
  • 입력: 복합체 구조 파일 (.pdb)
  • 출력: 예측 결합 자유 에너지 (ΔG kcal/mol), 온도 기반 Kd, 인터페이스 접촉 수, 비상호작용 표면 비율
  • 파이프라인에서의 의미:
    • Vina 점수(경험적 스코어)와 별개로, “구조 기반 인터페이스 특성”에서 나온 친화도 추정치를 제공
    • 후보 포즈/후보 펩타이드 간 상대 랭킹을 보강하는 추가 근거(특히 후속 GBSA/ML affinity 전 단계의 빠른 스크리닝용)
  • 설명
    • PRODIGY는 복합체 구조로부터 인터페이스 접촉 특징을 계산한다.
    • 계산된 특징을 기반으로 회귀 모델을 통해 ΔG를 예측한다.
    • 이는 도킹 스코어가 아니라 실험적 친화도 데이터에 맞춘 구조 기반 예측값이다.
    • 구조 정확도가 예측 성능에 직접적인 영향을 미친다.
    • → 이 구조가 실제 실험적으로 얼마나 강하게 결합할 가능성이 있는가?
  • 계산 과정
    • 복합체 구조(PDB)를 입력받아 다음을 수행
      • 구조에서 “결합 부위 특징(feature)”을 추출: 인터페이스 분석(접촉 잔기 수 (contact count), 비극성/극성 잔기 분포, 인터페이스 면적, 수소결합 관련 통계)
    • 회귀 기반 예측: 실험적 binding affinity 데이터로 학습된 회귀 모델 → ΔG (kcal/mol) 예측, Kd 추정
