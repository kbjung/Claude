1. 시스템 개요

1-1. 개요

  • AI 기반 단백질 결합 펩타이드 후보 예측 시스템
  • AI 기반으로 펩타이드-단백질 간 결합을 예측하고, 고결합력 후보 펩타이드를 도출하기 위한 파이프라인의 구성 계획
  • 전체 시스템은 펩타이드 생성, 구조 예측, 결합 평가 및 후보 선정의 과정을 포함하며, 각 단계는 자동화 및 통합 가능한 모듈 단위로 설계함

1-2. 목표

  • 단백질 타겟에 특이적으로 결합하는 펩타이드 후보군 도출
  • 전체 워크플로우 자동화 및 반복 실험에 최적화된 파이프라인 구축
  • 연구자 중심의 인터페이스 및 재사용 가능한 코드베이스 설계

3. 파이프 라인 구성 모델

1. 펩타이드 생성

  • 도구/모델: PepMLM
  • 역할: 생물학적으로 타당한 펩타이드 서열 생성 (MLM 기반)
  • 입력: 타겟 단백질 정보(구성 서열 문자열)
PepMLM
  • 핵심 개념
    • 표적 단백질의 서열/영역을 조건으로 펩타이드 후보 서열을 생성. (ESM-2 기반 MLM 파인튜닝)
    • 3D 구조를 직접 예측하진 않으며, 대규모 단백질 서열 학습에서 얻은 통계·공진화 패턴을 통해 구조·기능 제약을 간접 반영
  • 모델 학습 과정 설명(이해를 위한)
    • RCSB PDB 자료 중 단백질, 펩타이드 복합체 자료만 추출 후 전처리를 통해 약 1만개 샘플로 훈련
      • “[단백질 서열] + [펩타이드 서열]”로 입력
        • 단백질 특정 위치(양쪽 끝이 아닌)와 붙어있는 상태여도 “[단백질 서열] + [펩타이드 서열]” 인 한줄로 입력
      • 데이터 입력 후 훈련은 “[단백질 서열] + [MASK]”에서 가려진 부분(MASK)의 원래 서열을 예측하도록 훈련
    • 언어 모델 설명 예시
      • The capital of korea is Seoul
        • The capital of korea is [MASK]
        • 앞의 단어와 문장의 순서/구조 등으로 가려진 부분의 단어를 예측
          • 앞의 단어와 문장의 순서/구조 등 → 단백질 서열
          • 가려진 부분의 단어를 예측 → 펩타이드 서열 예측
        • 간접적으로 단백질 서열의 구조 등을 학습

2-1. 구조 예측

  • 도구/모델: ColabFold(AlphaFold-multimer v3)
  • 역할: 단백질–펩타이드 복합체의 3D 구조 예측
  • 입력: .fasta 또는 .a3m (단백질 + 펩타이드)
  • 출력: .pdb 구조 파일 + ipTM/pTM 점수 + ranking JSON
  • 메트릭: ipTM, pTM 점수 → 구조 신뢰도 판단(AlphaRank)
ColabFold(AlphaFold-multimer)
  • 핵심 개념
    • 단백질 복합체의 3D 구조를 예측하는 딥러닝 모델(AlphaFold-Multimer 기반)
  • 모델 설명
    • AlphaFold-Multimer: 여러 체인이 함께 접히는(co-folding) 복합체 3D 구조를 서열(+MSA/템플릿)로 예측하는 딥러닝 모델
    • ColabFold: AF/AF-Multimer를 쉽고 빠르게 돌리게 해주는 파이프라인(특히 MMseqs2 기반 MSA로 속도·접근성 ↑). 모델 정확도 자체는 대체로 기본 AF와 동급

2-2. 구조 튜닝

  • 도구/모델: OpenMM
  • 역할: 구조 예측(ColabFold) 결과 복합체 구조를 물리 기반으로 리파인(국소 안정화)하여 후속 평가(Vina/PLIP/PRODIGY)의 입력 구조 품질을 개선
  • 입력: 복합체 구조 파일(.pdb)
  • 출력: 튜닝된 구조 파일(.pdb) + 튜닝 로그(에너지 변화, 수렴 여부 등)
  • 메트릭(옵션): potential energy 변화(상대 비교/이상치 탐지), RMSD(튜닝 전후 구조 변화량), clash 감소 여부(품질 확인용)
OpenMM
  • 핵심 개념
    • OpenMM은 분자동역학(MD) 및 에너지 최소화(energy minimization)를 수행하는 물리 기반 시뮬레이션 엔진.
    • 이 단계의 목적은 “구조를 새로 예측”하는 것이 아니라, 예측된 PDB를 포스필드(force field)로 짧게 다듬어 물리적으로 더 자연스러운 상태로 만드는 것.
  • 수행 내용(파이프라인 기준)
    • 에너지 최소화: 원자 간 충돌(clash), 비정상적으로 가까운 접촉, 불안정한 기재(geometry)를 완화해 구조를 국소적으로 정리
    • 짧은 relaxation(옵션): 필요 시 아주 짧은 시간 범위에서 구조를 안정화(과도한 변형 방지를 위해 제한적으로 사용)

3-1. 도킹

  • 도구/모델: AutoDock Vina
  • 역할: 예측된 복합체 구조를 기반으로 결합 안정성(ΔG, kcal/mol) 평가
  • 입력: 리간드(pdbqt) + 리셉터(pdbqt) + 도킹 설정 (conf.txt)
  • 출력: 도킹 포즈(.pdbqt) + Vina 로그 텍스트(.txt) (binding affinity kcal/mol)

포즈 탐색 기능 평가 중

AutoDock Vina
  • 핵심 개념
    • 분자 도킹(Molecular Docking) 시뮬레이션 툴
    • 입력을 바탕으로 결합 가능한 포즈를 탐색, 각 포즈를 평가해 결합 에너지로 랭킹

3-2. 상호작용 분석

  • 도구/모델: PLIP
  • 역할: 도킹 구조 내 상호작용(수소결합, 소수성 등) 정량 분석
  • 입력: 도킹 결과 구조 (.pdb)
  • 출력: PLIP 리포트 파일(.xml)
  • 지표: 상호작용 수량 (결합 신뢰도 간접지표)
PLIP
  • 핵심 개념
    • 분자 간 결합 상호작용을 상세히 분석하는 툴.
    • 수소 결합, 소수성 상호작용 등 비공유 결합을 종류별로 식별. 식별된 상호작용의 개수를 정량적으로 파악.

3-3. 친화도 예측

  • 도구/모델: PRODIGY
  • 역할: 구조 기반 정밀 결합 친화도 예측 (ML 모델)
  • 입력: 복합체 구조 파일 (.pdb)
  • 출력: 결합 자유 에너지 (ΔG kcal/mol) 및 Kd 예측값
Prodigy
  • 핵심 개념
    • 3D 복합체 구조를 기반으로 결합 친화도(Kd)와 결합 자유 에너지(ΔG)를 예측하는 머신러닝 모델.
    • 결합 부위의 잔기 수, 비극성 표면적 등 물리적으로 해석 가능한 구조적 특징을 분석하여 결합력을 예측.

4. 최종 평가

  • 평가 방법 도입 로드맵
    • ipTM + Vina + PLIP + Pafnucy : 기존 조합 방식 (정확도 : B급)
    • [현재 방법] pTM + Vina + PLIP + PPI-Affinity : 개선된 방식 (정확도 : A급)
    • ipTM + Vina + PLIP + PRODIGY : 개선된 방식 (정확도 : A+급)
  • 여러 모델 조합방식은 흔함.
    • 동일한 구성은 드묾.
    • prodigy 비교적 최근에 많이 사용되기 시작함.
      • 2016~2018년까지는 연간 수십 건 인용 → 2020년 이후는 매년 수백 건 이상 인용으로 증가.
  • 도구/모델 : ipTM + vina + PLIP + PRODIGY
  • 전략
    • 각 평가 점수를 정규화(normalization) 후 가중 합산
      • 각 점수를 0~1 사이 고정 범위 스케일링(사전 정의된 최소/최대값 기준)으로 정규화한 후 가중 합산

        • Vina (25%): 두 번째로 높은 가중치. 좋습니다. Vina의 결합 에너지는 물리 법칙에 기반한 결합 안정성을 나타내는 강력한 지표입니다. PRODIGY를 보조하는 핵심적인 물리 점수로서 적절한 비중입니다.
        • PLIP (15%): 보조적인 가중치. 적절합니다. 상호작용의 '개수'는 결합의 '질'을 완벽히 대변하지는 못합니다. 중요한 상호작용 몇 개가 약한 상호작용 여러 개보다 나을 수 있기 때문입니다. 따라서 결합의 타당성을 뒷받침하는 보조 지표로서 좋은 가중치입니다.

      • 최적의 가중치를 찾는 과학적인 방법 🎯
        • 여기서 더 나아가 가중치에 대한 데이터 기반의 객관적인 근거를 만들고 싶으시다면, 이전에 설명해 드린 벤치마크 테스트를 활용하여 최적화를 진행할 수 있습니다.
        • 목표: Final_Score와 실제 실험값 사이의 상관계수를 가장 높게 만드는 가중치 조합을 찾는다.
        • 방법
          1. 데이터 준비: PDBbind와 같은 벤치마크 데이터셋에 대해 각 평가 모델(Vina, PRODIGY, PLIP, ipTM)의 점수를 모두 계산해 둡니다.
          1. 가중치 탐색 (Grid Search): 모든 경우의 수를 테스트하는 '격자 탐색(Grid Search)' 방법을 사용합니다.
            • 예를 들어, 각 가중치를 0.1 단위로 변경하며 모든 조합을 테스트합니다. (단, 합은 1.0이 되도록)
            • (w1, w2, w3, w4) = (0.4, 0.3, 0.2, 0.1)일 때의 상관계수 계산
            • (w1, w2, w3, w4) = (0.5, 0.2, 0.2, 0.1)일 때의 상관계수 계산
            • ... (수많은 조합 반복)
          1. 최적 조합 선택: 위 과정을 통해 계산된 수많은 상관계수 중 가장 높은 값을 보인 가중치 조합을 최종적으로 선택합니다. 이것이 바로 우리 데이터셋에서 최고의 성능을 내는 '경험적으로 검증된' 최적의 가중치가 됩니다.
      • 결론 및 제안
        • 옵션 1 (좋음): 현재 가중치를 그대로 사용합니다. 이미 논리적으로 충분히 타당하며, 파이프라인의 성능을 잘 대표할 가능성이 높습니다.
        • 옵션 2 (더 좋음): 벤치마크 테스트를 통해 최적의 가중치를 직접 찾습니다. 이 과정을 거치면 "우리의 파이프라인은 이 데이터셋에서 상관계수 0.85를 달성했으며, 이때의 최적 가중치는 Vina 20%, PRODIGY 55%...입니다" 와 같이 객관적인 근거를 바탕으로 파이프라인의 신뢰도를 주장할 수 있게 됩니다.
    • 상위 N개 구조를 최종 후보로 선정
