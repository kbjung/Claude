상세 내용

1. 펩타이드 생성

  • 도구/모델: PepMLM
  • 역할: 생물학적으로 타당한 펩타이드 서열 생성 (MLM 기반)
  • 입력: 타겟 단백질 정보(구성 서열 문자열)
    • PD-L1 으로 테스트(2025.09.02.화 ai회의 결과. pf 확인)

2. 구조 예측

  • 도구/모델: ColabFold(AlphaFold-multimer v3)
  • 역할: 단백질–펩타이드 복합체의 3D 구조 예측
  • 입력: .fasta 또는 .a3m (단백질 + 펩타이드)
  • 출력: .pdb 구조 파일 + ipTM/pTM 점수 + ranking JSON
  • 메트릭: ipTM, pTM 점수 → 구조 신뢰도 판단(AlphaRank)

3-1. 도킹

  • 도구/모델: AutoDock Vina
  • 역할: 예측된 복합체 구조를 기반으로 결합 안정성(ΔG, kcal/mol) 평가
  • 입력: 리간드(pdbqt) + 리셉터(pdbqt) + 도킹 설정 (conf.txt)
  • 출력: 도킹 포즈(.pdbqt) + Vina 로그 텍스트(.txt) (binding affinity kcal/mol)

3-2. 상호작용 분석

  • 도구/모델: PLIP
  • 역할: 도킹 구조 내 상호작용(수소결합, 소수성 등) 정량 분석
  • 입력: 도킹 결과 구조 (.pdb)
  • 출력: PLIP 리포트 파일(.xml)
  • 지표: 상호작용 수량 (결합 신뢰도 간접지표)

3-3. 친화도 예측

  • 도구/모델: PRODIGY
  • 역할: 구조 기반 정밀 결합 친화도 예측 (ML 모델)
  • 입력: 복합체 구조 파일 (.pdb)
  • 출력: 결합 자유 에너지 (ΔG kcal/mol) 및 Kd 예측값

4. 최종 평가

  • 평가 방법 도입 로드맵
    • ipTM + Vina + PLIP + Pafnucy : 기존 조합 방식 (정확도 : B급)
    • [현재 방법] pTM + Vina + PLIP + PPI-Affinity : 개선된 방식 (정확도 : A급)
    • ipTM + Vina + PLIP + PRODIGY : 개선된 방식 (정확도 : A+급)
  • 여러 모델 조합방식은 흔함.
    • 동일한 구성은 드묾.
    • prodigy 비교적 최근에 많이 사용되기 시작함.
      • 2016~2018년까지는 연간 수십 건 인용 → 2020년 이후는 매년 수백 건 이상 인용으로 증가.
  • 도구/모델 : pTM + vina + PLIP + PRODIGY

        • Vina (25%): 두 번째로 높은 가중치. 좋습니다. Vina의 결합 에너지는 물리 법칙에 기반한 결합 안정성을 나타내는 강력한 지표입니다. PRODIGY를 보조하는 핵심적인 물리 점수로서 적절한 비중입니다.
        • PLIP (15%): 보조적인 가중치. 적절합니다. 상호작용의 '개수'는 결합의 '질'을 완벽히 대변하지는 못합니다. 중요한 상호작용 몇 개가 약한 상호작용 여러 개보다 나을 수 있기 때문입니다. 따라서 결합의 타당성을 뒷받침하는 보조 지표로서 좋은 가중치입니다.

        • 여기서 더 나아가 가중치에 대한 데이터 기반의 객관적인 근거를 만들고 싶으시다면, 이전에 설명해 드린 벤치마크 테스트를 활용하여 최적화를 진행할 수 있습니다.
        • 목표: Final_Score와 실제 실험값 사이의 상관계수를 가장 높게 만드는 가중치 조합을 찾는다.
        • 방법
          1. 데이터 준비: PDBbind와 같은 벤치마크 데이터셋에 대해 각 평가 모델(Vina, PRODIGY, PLIP, ipTM)의 점수를 모두 계산해 둡니다.
          1. 가중치 탐색 (Grid Search): 모든 경우의 수를 테스트하는 '격자 탐색(Grid Search)' 방법을 사용합니다.
            • 예를 들어, 각 가중치를 0.1 단위로 변경하며 모든 조합을 테스트합니다. (단, 합은 1.0이 되도록)
            • (w1, w2, w3, w4) = (0.4, 0.3, 0.2, 0.1)일 때의 상관계수 계산
            • (w1, w2, w3, w4) = (0.5, 0.2, 0.2, 0.1)일 때의 상관계수 계산
            • ... (수많은 조합 반복)
          1. 최적 조합 선택: 위 과정을 통해 계산된 수많은 상관계수 중 가장 높은 값을 보인 가중치 조합을 최종적으로 선택합니다. 이것이 바로 우리 데이터셋에서 최고의 성능을 내는 '경험적으로 검증된' 최적의 가중치가 됩니다.
      • 결론 및 제안
        • 옵션 1 (좋음): 현재 가중치를 그대로 사용합니다. 이미 논리적으로 충분히 타당하며, 파이프라인의 성능을 잘 대표할 가능성이 높습니다.
        • 옵션 2 (더 좋음): 벤치마크 테스트를 통해 최적의 가중치를 직접 찾습니다. 이 과정을 거치면 "우리의 파이프라인은 이 데이터셋에서 상관계수 0.85를 달성했으며, 이때의 최적 가중치는 Vina 20%, PRODIGY 55%...입니다" 와 같이 객관적인 근거를 바탕으로 파이프라인의 신뢰도를 주장할 수 있게 됩니다.

여러 모델 조합 평가 방법

  • 실제로 많은 논문들이 단일 모델의 점수만으로 최종 결론을 내지 않고, 여러 평가 지표나 모델을 조합해 통합 평가함.
  • 각 도구는 장단점이 다르기 때문에 보완적 사용이 필요하기 때문.

논문에서 흔히 쓰이는 조합 사례

  1. 도킹 점수 + 결합 자유 에너지 보정
    • AutoDock Vina 같은 도킹 소프트웨어 → 1차 점수(ΔG)
    • 이후 MM/GBSA, MM/PBSA 같은 더 정밀한 계산으로 보정
    • 논문에서 "도킹 점수 + MM/GBSA로 재랭킹(re-ranking)" 패턴이 매우 흔함
  1. 도킹 점수 + 상호작용 분석(PLIP 등)
    • Vina 점수로 스크리닝 → PLIP/Discovery Studio 등으로 수소결합, 소수성 상호작용 개수 확인
    • 논문에서는 "상위 후보군 중 수소결합 ≥3개 형성한 리간드를 최종 선택" 식의 조건을 붙임
  1. 구조 기반 예측 + ML 기반 예측
    • PRODIGY(특징 기반 ML, 해석 가능) + PPI-Affinity(GNN 기반, 패턴 학습) 같이 서로 다른 방식의 모델 병행
    • 논문에선 "두 모델 모두에서 ΔG < -8 kcal/mol인 후보를 최종 선정" 같은 교차 기준을 씀
  1. 물리 기반 + 통계/AI 기반 점수 조합
    • 예: Vina (물리 기반), PRODIGY (구조 특징 기반), Deep Learning 모델 (패턴 기반) → 가중 평균/투표 방식으로 최종 점수화
    • 특히 최근엔 딥러닝 모델(Pafnucy, DeepBindRG 등)과 전통적 도킹 점수를 합쳐 hybrid pipeline 만드는 연구가 늘어남

실제 논문 예시 패턴

  • 약물 재창출(drug repurposing) 연구
    • Vina docking score + MM/GBSA + PLIP 상호작용 개수
  • 펩타이드 기반 억제제 설계 연구
    • AlphaFold로 복합체 구조 예측 → Vina로 안정성 평가 → PRODIGY로 친화도 예측 → PLIP으로 결합 모드 해석
  • 머신러닝 응용 연구
    • Docking 결과를 ML 모델(Pafnucy, DeepDTA 등)에 넣어 ΔG 예측값 보정

결론

만든 파이프라인 (pTM + Vina + PLIP + PRODIGY 조합)은 실제 연구자들이 쓰는 방식과 거의 같은 흐름이에요.

  • Vina = 빠른 안정성 지표
  • PLIP = 구조적 결합 근거
  • PRODIGY = 정밀 친화도 예측
  • pTM/ipTM = 구조 신뢰도 보정

논문에서도 흔히 다단계 필터링 + 다모델 조합 방식을 채택. 현재 접근법은 실제 연구 패턴과 일치
