PPI-Affinity vs PRODIGY

  • PPI-Affinity (패턴 기반 ML / Black Box)
    • 3D 구조 전체를 하나의 복잡한 패턴(그래프)으로 인식.
    • AI 모델(GNN/SVM)이 데이터 내 패턴을 스스로 학습하여 결합력 예측.
    • 예측 근거가 상대적으로 불투명하여 해석이 어려울 수 있음.
    • 원리: 단백질–단백질(또는 펩타이드–단백질) 3D 구조를 그래프(Graph) 형태로 변환 → 각 원자/잔기를 노드로, 상호작용(거리, 결합)을 엣지로 표현.
    • 모델 구조: GNN(그래프 신경망), SVM(서포트 벡터 머신) 같은 패턴 인식 모델이 그래프 데이터를 학습 → 훈련 데이터에 존재하는 결합 패턴을 통계적으로 학습.
    • 특징:
      • 장점: 데이터가 충분하다면 복잡한 상호작용 패턴까지 포착 가능 (예: 수소 결합, π-π stacking, van der Waals 등 다양한 복합 패턴).
      • 단점: 모델이 왜 특정 친화도를 예측했는지 해석하기 어려움 (블랙박스).
      • 한계: 훈련 데이터에 없는 새로운 구조적 패턴에는 취약할 수 있음 → 데이터 의존성이 높음.
  • PRODIGY (특징 기반 ML / White Box)
    • 3D 구조에서 물리적으로 의미가 명확한 특징들을 직접 계산.
    • 계산된 특징(예: 원자 접촉 수, 표면적)을 기반으로 결합력 예측.
    • 예측 근거가 명확하여 해석이 용이함.
    • 원리: 3D 구조(PDB)에서 물리적으로 해석 가능한 특징(feature) 직접 추출 → 예:
      • 결합 부위의 원자 접촉 수(contact count)
      • 극성/비극성 잔기 수
      • 비극성 표면적 (hydrophobic surface area)
      • 소수성 상호작용 정도
    • 모델 구조: 위와 같은 물리적 특징들을 입력 변수로 사용 → 머신러닝 회귀 모델(Random Forest, Linear Regression 등)을 통해 결합 친화도(Kd, ΔG) 예측.
    • 특징:
      • 장점: 왜 강하게 결합하는지 설명 가능 (예: 비극성 표면적 ↑ → 소수성 효과로 결합 안정성 ↑).
      • 단점: 미리 정의되지 않은 새로운 패턴은 반영하지 못할 수 있음.
      • 강점: 해석력(interpretability)과 일관성(consistency)이 매우 높음.

PRODIGY가 더 적절한 이유

  • 평가 방식의 일관성 (Consistency)
    • Vina, PLIP과 동일하게 3D 구조로부터 물리적으로 의미 있는 값을 계산.
    • '결합 부위 원자 수', '표면적' 등 물리적 특징을 직접 분석.
    • 파이프라인 전체가 '구조 → 물리적 특징 → 점수'라는 일관된 논리로 작동.
  • 해석의 용이성 (Interpretability)
    • 점수에 대한 물리적 근거(예: 비극성 표면적)를 바탕으로 설명 가능.
    • 후보 펩타이드가 '왜' 좋은지에 대한 높은 설명력 제공.
    • '블랙박스' 모델(GNN/SVM) 대비 해석이 매우 용이함.
  • 신뢰도와 안정성 (Robustness)
    • 상대적으로 간단하고 명확한 구조적 특징을 사용.
    • 입력 구조의 작은 부정확성에도 안정적인 예측 경향을 보임.
    • 훈련 데이터에 없는 새로운 형태의 구조에도 강건함(Robust).

모델별 계산 프로세스

  • PPI-Affinity (GNN/SVM 기반)
    • 입력: 복합체 3D 구조 (PDB 파일).
    • 프로세스:
      • 3D 구조를 그래프 또는 데이터 패턴으로 변환.
      • 변환된 데이터를 학습된 머신러닝 모델(GNN/SVM)에 입력하여 결과 예측.
    • 계산 속도: 수 초 이내 (매우 빠름).
  • PRODIGY (특징 기반)
    • 입력: 복합체 3D 구조 (PDB 파일).
    • 프로세스:
      • PDB 파일에서 결합 부위의 상호작용 원자 수, 표면적 등을 계산.
      • 계산된 구조적 특징들을 기반으로 결합 친화도 예측.
    • 계산 속도: 수 초 이내 (매우 빠름).

결론

  • PRODIGY 도입 시, 전체 평가 단계가 '해석 가능한 물리 기반' 컨셉으로 통일됨.
  • 파이프라인 결과의 신뢰도와 설득력을 크게 향상시킬 수 있음.
